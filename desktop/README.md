# TPT Vertex Desktop (Tauri)

The native desktop client for TPT Vertex. It wraps the web frontend
(`../frontend`) in a Tauri window and embeds the Rust geometry kernel directly,
so geometry evaluation and export run **offline-first with no server**
(see [ADR-0007](../docs/adr/0007-desktop-tauri.md)).

## Layout

```
desktop/
  package.json         # JS side: Tauri CLI + plugin API packages
  src-tauri/
    Cargo.toml         # Rust backend; depends on tpt-vertex-kernel & -manufacturing
    tauri.conf.json    # window, bundle, and updater configuration
    build.rs
    src/main.rs        # Tauri commands: evaluate_model, export_step_text
    icons/             # app icons (populate before bundling)
```

`src-tauri` is intentionally **excluded from the root Cargo workspace** so the
desktop app builds and releases on its own cadence; it still references the
kernel crates by relative path.

## Prerequisites

- Rust toolchain (stable)
- Node.js LTS
- Platform WebView + build tooling per the
  [Tauri prerequisites](https://tauri.app/start/prerequisites/):
  - **Windows**: WebView2 (preinstalled on Win11), MSVC build tools
  - **macOS**: Xcode command line tools
  - **Linux**: `webkit2gtk`, `librsvg`, `libayatana-appindicator`, etc.

## Develop

```bash
cd desktop
npm install
npm run dev      # starts the Vite frontend and opens the native window
```

## Native capabilities

- **File system access**: open/save local project files via
  `@tauri-apps/plugin-dialog` + `@tauri-apps/plugin-fs`.
- **Offline local kernel**: `evaluate_model` and `export_step_text` commands run
  the embedded `tpt-vertex-kernel`/`-manufacturing` crates in-process.
- **Auto-update**: `tauri-plugin-updater` checks the endpoint configured in
  `tauri.conf.json` (`plugins.updater`). Replace the `pubkey` placeholder with a
  key generated by `npm run tauri signer generate`.
- **Cloud handoff**: the same frontend opens a cloud project on demand; the
  desktop app never requires the server for local work.

## Package & sign

```bash
npm run build    # produces installers under src-tauri/target/release/bundle/
```

Signing/notarization is configured per OS before distribution:

- **Windows**: Authenticode certificate (`bundle.windows.certificateThumbprint`
  / `signCommand`).
- **macOS**: Developer ID Application certificate + notarization
  (`APPLE_CERTIFICATE`, `APPLE_ID`, `APPLE_PASSWORD` env in CI).
- **Linux**: AppImage/deb signed via the distribution's tooling.

Updater artifacts are signed with the Tauri updater private key
(`TAURI_SIGNING_PRIVATE_KEY`); the matching public key lives in
`tauri.conf.json`.

CI packaging runs `npm run build` on a Windows, macOS, and Linux runner and
uploads the signed bundles as release assets.
