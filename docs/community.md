# Community & Support

TPT Vertex is open-source (dual **MIT OR Apache-2.0**) and built for a global
community of hardware makers. Here's how to get help and get involved.

## Where to talk

- **GitHub Discussions** — questions, ideas, show-and-tell, and RFCs. This is the
  primary asynchronous forum.
- **GitHub Issues** — bug reports and concrete feature requests. Use the issue
  templates.
- **Chat (Discord/Matrix)** — real-time community chat. (Invite link to be added
  at launch — see the repository README.)

> The exact channels are finalized at launch; the repository README always links
> the current, canonical destinations.

## Getting help

1. Search existing Issues and Discussions first.
2. For usage questions, open a Discussion with a minimal description and, if
   possible, a small sample model or steps to reproduce.
3. For suspected bugs, open an Issue using the bug template and include your
   platform, browser/desktop version, and reproduction steps.

## Ways to contribute

- Report bugs and propose features.
- Improve documentation (this `docs/` directory).
- Triage and reproduce issues.
- Contribute code — see [Contributor Onboarding](contributor-onboarding.md) and
  [CONTRIBUTING.md](../CONTRIBUTING.md).
- Build and share plugins using the [plugin interface](plugin-api.md).

## Code of Conduct

Participation is governed by our
[Code of Conduct](../CODE_OF_CONDUCT.md). Be kind, be constructive, assume good
faith.

## Governance (v0.1)

The project is currently maintained by TPT Solutions with community
contributions. Significant, architecturally-significant decisions are recorded as
[ADRs](adr/README.md) and discussed in the open before acceptance. A broader
governance model will evolve with the community.
